Web Dashboard

Starting from the release 2.8.8, the library has its own Web Dashboard that can be used to monitor and profile the performance of running WindFlow applications. Information related to how to start and configure the dashboard can be found in the GitHub readme of the library.

Once running, the dashboard opens a REST server on a port (8080 by default) of the machine. To avoid performance conflicts between the dashboard threads and the ones of the running WindFlow applications, the dashboard can be executed on a different machine with respect to the ones used to run the WindFlow applications to be monitored.

The user can inspect the statistics of each running operators and of each of its internal replicas. Statistics are updated every second, and report the number of received/transmitted inputs/outputs, processing time, and so forth. Figure below shows the report page of the application.

The last screenshot below shows the report of statistics opened for one operator and the historical statistics (input and output rate per second over the whole execution) for a different operator opened in the same web page.

The Web Dashboard aims at being a useful tool for developers of WindFlow applications, because it helps in understanding what are the performance bottlenecks of the application and to profile its execution throughout the execution.
